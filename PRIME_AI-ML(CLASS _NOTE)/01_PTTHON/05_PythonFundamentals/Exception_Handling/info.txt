try
Definition :-try contains the code that may cause an error.
Purpose :- Test risky code without stopping the program.

except
Definition :-except handles the error that occurs in the try block.
Purpose :-Prevents program crash and shows a friendly message.

else
Definition :-else runs only if no exception occurs in try.
Purpose :-Keep success code separate from error handling.

finally
Definition :- finally runs always, whether an error occurs or not.
Purpose :- Used for cleanup actions (close file, release resources).
